User Data & Privacy

Last updated: 21 April 2026

Principle: Users are identified by name/email for account access only. All health data is stored and queried by user_id alone — never by name or email.

Data Classification

TierLabelDefinition
1IdentityDirectly identifies user (email, name)
2HealthGDPR Article 9 — highest protection
3BehaviouralUsage patterns, content read
4SystemInfrastructure/auth only

Data Inventory

Users Table (Identity + Auth + Health)

  • Tier 1: email, name
  • Tier 2: birthday, last_period_start, cycle_length, period_length, contraception_type
  • Tier 4: encrypted_password, admin, OAuth UIDs

Symptom Logs (Health)

  • mood, energy, sleep, physical, mental, pain, cravings, discharge
  • temperature, weight
  • notes (free text — can contain PII)
  • sexual_intercourse

Cycle Entries (Health)

  • phase (menstrual/follicular/ovulation/luteal)
  • season_name (winter/spring/summer/autumn)
  • cycle_day, period_start, period_end

Target Architecture

Separate health data into its own table:

  • users — Identity + Auth only
  • user_health_profiles — All health columns, linked by user_id

Migration Priorities

PriorityActionEffort

Compliance

  • Play Store: Data safety form needed
