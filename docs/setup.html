Local Setup

Prerequisites

  • Ruby 3.4.7 — use rbenv or asdf
  • PostgreSQL running locally
  • Node.js (for Tailwind build watcher)

Installation

1

Clone and install dependencies

git clone git@github.com:Tich-Labs/season.git
cd season
bundle install
2

Add the master key — get this from the team password manager. Never commit it.

echo "YOUR_MASTER_KEY" > config/master.key
3

Create the database, load schema, and seed phase content

bin/rails db:create db:schema:load db:seed
4

Start the development server

bin/dev

Architecture

Auth

Custom cookie-based Authentication concern in app/controllers/concerns/authentication.rb. Devise is used only for password recovery emails. Sessions are encrypted cookies with a 7-day expiry. Do not re-include the Authentication concern in subclasses — it is included once in ApplicationController.

Cycle Logic

CycleCalculatorService (app/services/cycle_calculator_service.rb) is the single source of truth for phase, season, cycle day, and calendar colour data. Nothing else calculates phases — always go through this service.

Phase Content

CyclePhaseContent model, seeded for en/de × 4 phases. Stores superpower, mood, sport, nutrition, and care text per phase and locale.

i18n

English default, German available. All user-facing strings must go through t() helpers — never hardcode English text in views. Locale files: config/locales/en.yml and config/locales/de.yml.

Admin

admin/ namespace, gated by User#admin? boolean. Covers users, inbox (feedback/bugs/support), and launch signups.

PWA

Mobile-first, 390px base, max-w-[430px] layout container throughout. Turbo Native (iOS/Android) is on the roadmap but not yet implemented.

Codebase Structure

Controllers (29 total)

  • ApplicationController + Authentication concern — base auth
  • HomeController — landing, loader, countdown, welcome
  • SessionsController, RegistrationsController — login / signup
  • OnboardingController — 11-step onboarding flow
  • CalendarController — monthly / weekly / appointments
  • CalendarEventsController — CRUD for events
  • DailyViewController — day detail view
  • TrackingController — period entry
  • SymptomsController, SuperpowersController, StreaksController — tracking
  • SettingsController — all settings routes
  • PasswordsController, OmniauthController, InvitesController — auth utilities
  • LaunchSignupsController, LaunchController — waitlist + countdown
  • FeedbacksController, LegalController, PWAController
  • Admin::UsersController, Admin::InboxController, Admin::LaunchSignupsController

Models (12 total)

  • User — accounts, auth, health profile
  • CycleEntry — daily cycle tracking
  • CyclePhaseContent — phase content (en/de)
  • CalendarEvent — calendar events
  • SymptomLog — daily symptom logs
  • SuperpowerLog — daily superpower logs
  • Streak — tracking streaks
  • Reminder — user reminders
  • Feedback — user feedback (feedback / bug_report / support)
  • LaunchSignup — waitlist email signups
  • Current — request-scoped current user

Database Rules

  • PostgreSQL only — no SQLite in any environment
  • Reversibility — every migration must be reversible (change or explicit up/down)
  • No downtime migrations — avoid destructive operations without a two-step deployment
  • DB-level constraints — use null constraints and defaults at the database level, not just in Rails
