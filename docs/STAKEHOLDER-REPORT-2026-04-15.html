Stakeholder Report

Original date: 15 April 2026 — Updated: 26 April 2026

This is a non-technical summary of the current state of Season V2. For deeper technical context see the Project Overview.

Summary

Season V2 is feature-complete. All six in-scope milestones are built and deployed. The app is running live on Render, serving invited users. One item remains before a broader launch: adding OAuth API keys (Google, Facebook, Apple) to the production environment. No code changes are required for that step.

What is Built

  • Personalised invite flow for 150 existing V1 users
  • Full auth: signup, login, social login (pending OAuth keys), password reset with branded email
  • 11-step onboarding collecting cycle length, contraception, food preferences, and reminders
  • Calendar (monthly + weekly + appointments) colour-coded by cycle phase
  • Daily view with phase insights, symptom summary, tips, and events
  • Symptom log (mood, physical, mental, notes, temperature, weight)
  • Phase education library (4 phases, EN + DE)
  • Superpowers and streaks
  • Email reminders: morning summary, period reminder, birth control reminder — all with user-controlled settings
  • Settings: profile, calendar preferences, notification toggles, subscriptions
  • Admin panel: user list, feedback inbox, waitlist with CSV export
  • Launch page, waitlist signup, legal pages (terms + privacy)

What is Blocking Launch

One item only: OAuth credentials for Sign in with Google, Facebook, and Apple. The code is built and deployed. The API keys need to be added to the Render environment dashboard.

What is Planned Post-Launch

  • Stripe paywall (M8)
  • Hotwire Native wrapper for App Store distribution
  • Calendar sync (iCal / Google Calendar)
  • Accessibility improvements (ARIA for cycle picker)
  • i18n: extract remaining hardcoded onboarding strings
